GitHub Pages 活用ガイド

README を目次にして、専用Webページへ案内する手順書
開発グループ　廣田 ／ ver.1.0

🌷 このガイドについて

このガイドでは、GitHub Pages を使って HTML資料を社内共有する方法を、初めての方にも分かりやすく説明します。

ゴールは、README ページを「目次」として活用し、各資料の専用Webページへ簡単にアクセスできる仕組みを作ることです。

所要時間は、約 20〜30分。コーヒー☕を片手に、ゆっくり進めてください。

📑 目次

  1. GitHubアカウントを準備する
  2. リポジトリを作成する
  3. HTMLファイルをアップロードする
  4. GitHub Pages を有効化する
  5. README を目次として整える
  6. 部内メンバーへ共有する
  7. 困った時のTIPS集
1

GitHubアカウントを準備する

まずは GitHub のアカウントを作成します。すでにお持ちの方は次へ進んでください。

手順

  1. https://github.com/ にアクセス
  2. 右上の Sign up をクリック
  3. メールアドレス・パスワード・ユーザー名を入力
  4. 認証メールを確認して登録完了
ユーザー名はURLの一部になります。会社名や本名を含む分かりやすい名前がおすすめ（例: sugiyasu-hirota）。後から変更可能ですが面倒なので慎重に。
2

リポジトリを作成する

資料を入れる「箱」を作ります。リポジトリ = ファイル置き場 と覚えればOK。

手順

  1. GitHub 画面右上の + アイコン → New repository
  2. Repository name: 任意の名前（例: dev-docs）
    全部小文字＆ハイフン区切りがおすすめ。日本語や大文字混在はURL不具合の元。
  3. Public を選択（無料でPages公開できるのは★Public★のみ）
  4. Add a README file にチェック
  5. Create repository をクリック
Public（公開）を選ぶと、URLを知っていれば誰でも見られる状態になります。社外秘の資料を扱う場合は、必ず情シスに相談してください。
3

HTMLファイルをアップロードする

公開したいHTML資料を、フォルダにまとめてアップします。

推奨フォルダ構成

リポジトリ/
├── README.md          ← 目次ページ（次のステップで作成）
└── AdminDocs/         ← 資料の保管場所（閲覧者から見えない様に階層を下げている）
    ├── 資料1.html
    ├── 資料2.html
    └── images/        ← 画像があれば（基本は無いレベルで良いと思う）
        └── figure.png

手順

  1. リポジトリ画面で Add file → Upload files
  2. HTMLファイルや画像をドラッグ&ドロップ
  3. 下部の Commit changes をクリック
ファイル名は 半角英数字＆アンダースコア を推奨。
❌ NG例: 業務改善 (最新版).html
✅ OK例: 業務改善_v2.html または kaizen_v2.html
スペースや記号（！？など）はURL不具合の原因になります。

フォルダの作り方

GitHubでは「フォルダだけを単独で作る」ことはできません。
ファイルアップロード時に、ファイル名の前にフォルダ名を付けて作成します:

例: docs/資料1.html

これで docs フォルダが自動生成され、その中にファイルが配置されます。

4

GitHub Pages を有効化する

これが今回の主役！リポジトリのHTMLを、Webサイトとして公開するスイッチです。

手順

  1. リポジトリ画面の Settings タブをクリック
  2. 左メニューの Pages をクリック
  3. Source: Deploy from a branch を選択
  4. Branch: main / / (root) を選択
  5. Save をクリック
  6. 1〜3分待つ（コーヒーブレイク☕）
  7. 画面上部に緑チェック✅と共に公開URLが表示される

🎉 公開URLの例

https://<ユーザー名>.github.io/<リポジトリ名>/

このURLが、誰でも閲覧できるWebサイトのアドレスになります。

Pagesの有効化はリポジトリごとに1回だけ必要です。一度設定すれば、以降ファイルを更新するたびに自動で再公開されます。
5

README を目次として整える

閲覧者が最初に見るページです。各資料へのリンクを並べた「案内板」を作りましょう。

手順

  1. リポジトリトップで README.md をクリック
  2. 右上の鉛筆アイコン✏️をクリック
  3. 以下のテンプレートをコピペして編集
  4. Preview タブで見た目を確認
  5. Commit changes をクリック

テンプレート（コピペ用）

# 開発グループ 資料サイト

このページは、開発グループの技術資料を公開する目次です。
下記リンクから各資料をご覧いただけます。

---

## 📄 公開資料

### 👉 [資料1のタイトル](https://ユーザー名.github.io/リポジトリ名/docs/資料1.html)

資料1の概要を1〜2行で説明します。

### 👉 [資料2のタイトル](https://ユーザー名.github.io/リポジトリ名/docs/資料2.html)

資料2の概要を1〜2行で説明します。

---

## ご利用にあたって

- 上記リンクから資料をご覧ください
- 資料は随時更新されます
- 表示が古い場合は `Ctrl + F5` で再読み込みしてください

Markdownの基本記法

書き方意味
# 文字大見出し
## 文字中見出し
### 文字小見出し
[表示文字](URL)クリックできるリンク
- 文字箇条書き
---横線
編集画面の上部にある Preview タブを押すと、装飾された状態を確認できます。コミット前にチェックする習慣をつけましょう。
6

部内メンバーへ共有する

完成したサイトのURLを、部内メンバーに案内します（REDEMEから勝手に見るとは思う）。

共有時に伝えるべきこと

  • 📍 URL: https://ユーザー名.github.io/リポジトリ名/
  • 📖 使い方: トップページが目次なので、各資料リンクをクリック
  • ⚠️ 注意: 表示が古い時は Ctrl + F5 でリロード

共有メッセージのサンプル　・・・　あんまり必要ないかな

件名: 【開発グループ】資料サイト公開のご案内

お疲れさまです。
開発グループの資料を以下のサイトで公開しました。

▼ 資料サイト
https://ユーザー名.github.io/リポジトリ名/

トップページが目次になっていますので、
各資料のリンクをクリックしてご覧ください。

ご不明点があればお気軽にご連絡ください。
ブックマーク登録を推奨すると、何度も訪れてもらいやすくなります。チャットツールがあるならピン留めも◎

困った時のTIPS集

よくあるトラブルと対処法をまとめました。

Q. リンクをクリックしたら404エラーが出る

原因の多くは「URLとファイル名の不一致」です。以下を確認してください:

  • READMEに書いたURLと、実際のファイル名が完全一致しているか（大文字小文字も区別）
  • フォルダ名が正しいか
  • ファイル名にスペースや特殊記号が含まれていないか

Q. ファイルを更新したのに反映されない

  1. Actions タブでビルド状況を確認（緑✅になっているか）
  2. ブラウザで Ctrl + F5 でスーパーリロード
  3. シークレットモードで開いて確認

Q. ソースコードが表示されてしまう

URLが github.com になっている可能性があります。
正しいURLは ユーザー名.github.io で始まります。

NG（コード表示）github.com/ユーザー名/リポジトリ名/...
OK（サイト表示）ユーザー名.github.io/リポジトリ名/...

Q. ファイル名にスペースが入っていてリンクが壊れる

ファイル名を半角英数字＆アンダースコアにリネームするのが最良の解決策です。

Q. README編集画面でMarkdownが装飾されない

編集画面の Code タブは生のテキスト表示です。
隣の Preview タブを押すと装飾された状態が見られます。
